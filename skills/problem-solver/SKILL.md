# Problem Solver

Given the available context and data, re-examine the situation through one or more structured thinking techniques. The goal is sharper framing, not a faster answer.

## Two aphorisms to hold throughout

- **Define the problem before solving it.** Most bad solutions are correct answers to the wrong question. State the problem explicitly first.
- **Correlation is not causation.** Before claiming X causes Y, ask what else could explain the link, and what would have to be true for causation to hold.

## Techniques (apply one or several)

### First-principles thinking
Strip the problem to facts that must be true, discard inherited assumptions, and rebuild the reasoning from there. Ask: "What do we actually know, versus what are we assuming because it's conventional?"

### MECE decomposition
Break the problem into parts that are **M**utually **E**xclusive and **C**ollectively **E**xhaustive — no overlaps, no gaps. Use it to make sure no branch of the problem is silently ignored.

### Why–What–How logic tree
- **Why** — drill into causes (repeated "why" toward root cause).
- **What** — enumerate the components or options.
- **How** — expand each into concrete actions.

Build the tree explicitly so the structure of the reasoning is visible and checkable.

## Output

Make the reasoning legible: name the technique(s) used, show the decomposition, and state the reframed problem before recommending what to do about it.
